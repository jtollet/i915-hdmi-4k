## EXPLANATION: Why 201ms?

The printk messages measure AROUND the poll, not the exact success moment:

[6.452101] printk "started"  ← Before poll
    ↓
  [Poll starts, checks every 1ms]
    ↓
  ~6.650s: Scrambling = TRUE  ← SUCCESS (within 200ms!)
    ↓
  poll_timeout_us() returns 0
    ↓
[6.653498] printk "finished" ← After poll + overhead

Overhead sources:
1. poll_timeout_us() tolerances (can slightly exceed)
2. printk() execution time
3. Kernel scheduling latency
4. Last check cycle completion
