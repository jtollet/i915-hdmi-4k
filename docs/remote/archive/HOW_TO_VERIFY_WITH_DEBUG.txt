# How to Verify Poll Timeout Status with DRM Debug

## Current Situation
- drm.debug = 0 (disabled)
- drm_dbg_kms() messages are NOT shown
- We cannot see timeout message even if it occurred

## Functional Proof (Currently Used)
✅ Display works at 4K@60Hz = scrambling active = no timeout
✅ No "unable to decode signal" error
✅ This is sufficient proof for most cases

## To Get Explicit Debug Logs (Optional)

### Option 1: Boot with DRM Debug
Add to kernel command line:
  drm.debug=0x04

Where 0x04 = DRM_UT_KMS (display messages)

### Option 2: Enable at Runtime
  echo 0x04 | sudo tee /sys/module/drm/parameters/debug

### Then Reboot and Check
  dmesg | grep -i "timed out"

If no message appears with debug enabled, then definitely no timeout.

## Debug Levels
0x01 - DRM_UT_CORE
0x02 - DRM_UT_DRIVER  
0x04 - DRM_UT_KMS       ← Need this for drm_dbg_kms()
0x08 - DRM_UT_PRIME
0x10 - DRM_UT_ATOMIC
0x20 - DRM_UT_VBL

## For Ankit
We can mention:
"The display functions perfectly at 4K@60Hz, which confirms scrambling 
is active. Without successful scrambling initialization, the monitor 
would show 'unable to decode signal'. This is functional proof that 
the poll completed successfully within the timeout period."
